## 6. 데이터 흐름 (학교 업무 비유)

### 단계별 비유

| 단계 | 앱 동작 | 학교 업무 비유 |
|------|---------|---------------|
| 1. 로그인 | userId 발급 | 내 책상(서랍) 배정 |
| 2. 학급 설정 | classes 생성, students + accessCode | 출석부 작성, 학생 스티커 부여 |
| 3. 경기 설정 | games 생성 (날짜, 모드, 종목) | 경기 일지 작성 |
| 4. 기록 입력 | records 저장 (학생별 점수) | 기록장에 한 줄씩 기록 |
| 5. 성장 조회 | accessCode로 records 검색 | 스티커 번호로 기록 찾기 |

### 핵심 개념

```
모든 기록에 학생 스티커(accessCode)가 붙어있어서,
나중에 그 스티커로 검색하면 해당 학생의 모든 기록을 한번에 찾을 수 있음
```

## 7. 반응형 레이아웃

### 데스크톱 (1024px+)
- 사이드바 고정 표시
- 메인 영역 넓게 사용
- 학생 카드 5열 배치

### 태블릿 (768px ~ 1023px)
- 사이드바 접기/펼치기 가능
- 학생 카드 3열 배치

### 모바일 (767px 이하)
- 사이드바 숨김 (햄버거 메뉴)
- 학생 카드 2열 배치
- 탭 스와이프 전환
